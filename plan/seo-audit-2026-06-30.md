# SEO Audit — makecepeit.com (2026-06-30)

**SEO Health Score: 81 / 100** (B+). Strong technical + AI-readiness foundation; the main risk is thin/templated content at scale across the 842 programmatic pages.

**Business type:** Freemium SaaS tool (receipt generator) + large programmatic-SEO content layer.
**Scope:** 900 indexable URLs — 348 brands, 275 examples, 219 receipt-help, 42 templates, 16 core/static.

## What's already excellent
- Clean single-hop redirects: `http→https`, `non-www→www`, trailing-slash normalization (all 308).
- All security headers (HSTS w/ preload, nosniff, referrer-policy, permissions-policy, CSP frame-ancestors).
- robots.txt + sitemap correct; `/admin`, `/account`, `/login`, `/api/` disallowed.
- Rich schema: home has WebApplication, Organization, FAQPage, HowTo, Offer; detail pages have BreadcrumbList, FAQPage, ItemList.
- `llms.txt` detailed and well-structured (top-tier AI readiness; addresses the fraud/legitimacy angle).
- 100% image alt coverage (incl. 348 brand logos); clean single-H1 + semantic H2 outline.

## Findings by priority

### HIGH
- **H1. Thin / near-duplicate programmatic content.** `/brands/*` ~283 words, `/examples/*` ~311 words (×623 pages), shared structure. Risk: thin/doorway classification → index bloat, crawl waste, soft filtering. `/receipt-help/*` (~496w) are fine. Fix: add unique brand/example-specific copy + varied FAQ, target 450+ unique words. Files: `app/brands/[slug]/page.tsx` + `lib/brands.ts`, `app/examples/[slug]/page.tsx` + `lib/examples.ts`.
- **H2. Missing OG/social images on ~390 pages.** `/brands/*` and `/templates/*` declare `twitter:card: summary_large_image` but ship no `og:image`. Lower social CTR. Fix: add `openGraph.images` (or apply default `opengraph-image`) in those `generateMetadata`.

### MEDIUM
- **M1. Title tags truncating.** `/brands` (70), `/receipt-help` (68), `/templates` (67), examples detail (93 chars). Trim `| Makecepeit` suffix on long titles; shorten example title pattern.
- **M2. Listing pages missing ItemList + BreadcrumbList schema.** `/templates`, `/examples`, `/receipt-help`, `/blog` only carry Organization + WebSite.
- **M3. `/examples` list page heavy** — 1.17 MB HTML / ~4,000 words in one response. Hurts LCP/CLS + crawl. Render fewer cards / use existing `/examples/page/N` pagination.
- **M4. Blog effectively empty** (index only, no posts). Biggest untapped content lever for an SEO-led launch.

### LOW
- **L1.** Measure real CWV via PageSpeed Insights / CrUX (not measurable from audit tooling here). Heavy examples page is the likely weak spot.
- **L2.** `http://makecepeit.com` takes 2 redirect hops (minor).

## Recommended order
1. H2 (OG images) — quick, in-code, lifts CTR.
2. M1 (titles) + M2 (listing schema) — small changes, broad impact.
3. H1 (thin content) — the big one; brands first (highest volume).
4. M4 (blog) — ongoing.

_Performance (CWV) not scored — no Lighthouse/PSI access during audit. All other findings from live production + codebase._
